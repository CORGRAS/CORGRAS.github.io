Solving problems when installing modules

We have collected solutions to the most common problems.

During / after installing some modules or modifiers in Opencart, you may encounter errors.

Solution to the problem:

Download and install the add-on Localcopy OCMOD

After installing this add-on, go to the Modifiers section and click "Update modifiers". After that, you can install other modules.

There are several possible problems and solutions:

1. The template cache was not reset - you need to update the cache [screenshot]

2. You used to edit template files through the admin panel in the "Theme Editor" section, then this file was saved in the database and it is no longer affected by ocmod modifiers, this is an Opencart problem 3. You need to install a fix [download] and update the modifications in the admin panel.

3. Your template is based on TPL files (*.tpl), while in Opencart 3 templates should be in TWIG format (*.twig). The problem is that ocmod modifiers do not affect TPL files-all changes must be made to these files manually.

There is a bug in the official Opencart 3.0.3.5 - 3.0.3.6 builds where modifiers don't work with twig files due to an error in the standard modifier /system/modification.xml - the second operation from this modifier does not work.

The problem is available on the official github and a solution has been proposed https://github.com/opencart/opencart/blob/3.0.x.x_Maintenance/upload/system/modification.xml

We made a fixed modifier based on these recommendations. It will replace the file /system/modification.xml to the correct one.

Solving the problem:

  1. Make a backup copy of your file /system/modification.xml
  2. Download the fix modifier [download] and install it in your admin panel
  3. Update the modifier cache
  4. Update the cache of twig templates

If the module does not respond to your actions, whether it is clicking on a button or loading a map, selecting a date, or any other action that is included in the module's functionality, then there is most likely a conflict in the JS scripts.

Press F12 on the problem page in the browser-the webmaster panel opens, where you can see conflicts in scripts on the Console tab, if any. This action is available in Chrome, Opera, and Firefox browsers.

Error 500 after installing the module means that some critical error occurred in php.

To determine the problem, you can:

1. Open the logs/error.log file on your server. At the end of this file, you will see all the latest critical errors.

2. Open in the admin panel Tools > Error logs - errors are displayed in this place, but not always.

3. Enable displaying php errors on the screen-add the following code to the end of the file .htaccess:

php_value display_errors 1

php_value display_startup_errors 1

php_value error_reporting E_ALL

If the module is a single XML file, then its extension must be .ocmod.xml

If the module is ocmod.zip archive, then you don't need to unpack it, but install it as it is. This archive must contain the upload folder (it can be empty), and it can also contain modification files: install.xml, install.php, install. sql. There should be no other files in the archive root.

To resolve this error, you need to delete the old version of the modification in the section before installing it Extensions > Modifications.

If you don't have such a module, but the ID matches, then you need to change the parameter value code in the XML modifier to install, make this parameter unique by adding a few characters.

Number of characters in ocmod.xml the value of the file must not exceed 65535

To resolve the error, you need to split the modification xml file into several parts. The main thing is to remember to set a unique value for each part in <code>, ou can add the digits 1,2,3... to the current value as part identifiers.

Another solution is to change the type of fields where modifications are stored in the 'oc_modification' table in the database. You need to run the following SQL query::
ALTER TABLE oc_modification CHANGE xml MEDIUMTEXT CHARACTER SET utf8 COLLATE utf8_general_ci NOT NULL ;
